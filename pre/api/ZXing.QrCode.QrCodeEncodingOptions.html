    DisableECI

    Explicitly disables ECI segment when generating QR Code That is against the specification of QR Code but some readers have problems if the charset is switched from ISO-8859-1 (default) to UTF-8 with the necessary ECI segment. If you set the property to true you can use UTF-8 encoding and the ECI segment is omitted.

    QrCompact

    Specifies whether to use compact mode for QR code (type Boolean, or "true" or "false" Please note that when compaction is performed, the most compact character encoding is chosen for characters in the input that are not in the ISO-8859-1 character set. Based on experience, some scanners do not support encodings like cp-1256 (Arabic). In such cases the encoding can be forced to UTF-8 by means of the CharacterSet encoding hint.
